Week 8 — K-Nearest Neighbors & Distance Metrics¶

Methods covered:

  1. KNN Classification — majority vote among k nearest neighbors
  2. Distance Metric Comparison — Euclidean (L2), Manhattan (L1), Chebyshev (L∞)
  3. Cross-validated k selection — StratifiedKFold, k ∈ {1, 3, …, 29}

Datasets: Fraud Detection (300K sample) · Credit Approval · Loan Default
Primary metric: AUC-ROC
Scaling: StandardScaler mandatory — KNN is distance-based, unscaled features dominate

Section 1: Fraud Detection Dataset¶

KNN applied to the class-imbalanced fraud dataset (0.58% fraud rate).
All search phases use a 300K stratified sample — KNN inference is O(N·d),
making full 1.3M-row evaluation computationally prohibitive across 45 k×metric configs.
Cosine distance is excluded: designed for text/embedding spaces (direction-only similarity),
not for tabular financial features where magnitude differences carry predictive signal.

Fraud Detection — KNN Analysis¶

Best configuration: k=29, Euclidean distance (CV AUC: 0.8990 → Test AUC: 0.8791)

All 15 k values showed monotonically increasing AUC — the curve had not peaked at k=29. Extending the range further is unlikely to close the gap with Milestone 1 models; the issue is structural, not a tuning problem.

Why KNN underperforms (−0.1029 vs Wk1 Poly-LR):

  1. Class imbalance dominates neighbor voting — with only 0.58% fraud, k=29 neighbors are overwhelmingly legitimate transactions regardless of proximity.
  2. No feature weighting — KNN assigns equal importance to all 23 features. Logistic Regression and Random Forest learn that amt and transaction patterns carry disproportionate predictive signal; KNN cannot.
  3. Curse of dimensionality — in 23-dimensional space, neighbor distances become nearly uniform, weakening the core assumption of instance-based learning.

Distance metric: Euclidean outperformed Manhattan and Chebyshev. The squared penalty on large coordinate differences likely captures outlier transaction amounts — a key fraud signal — more effectively than L1 or L∞.

Overfitting check: CV AUC (0.8990) exceeds Test AUC (0.8791) by 0.020 — a modest generalization gap consistent with expected variance, not a red flag.

Marketing context: In customer churn prediction, KNN's equal-weight limitation similarly hurts when recency and purchase frequency dominate. Feature-weighted models are preferred for high-value retention campaigns.

Credit Approval — KNN Analysis¶

Best configuration: k=29, Chebyshev distance (CV AUC: 0.8404 → Test AUC: 0.9056)

Reversed CV vs Test gap: Test AUC (0.9056) exceeds CV AUC (0.8404) by 0.065 — the inverse of the typical pattern. Chebyshev's L∞ metric (maximum coordinate difference only) may interact favorably with the specific 80/20 stratified split. The credit dataset's relatively clean class separability makes this outcome plausible.

Why Chebyshev wins here: Credit applicant profiles likely have one dominant differentiating feature (e.g., total income or employment duration) that Chebyshev isolates by ignoring all other coordinate differences. This acts as an implicit feature selection mechanism.

KNN underperforms Week 5 SVM by 0.0943 AUC — SVM's margin maximization exploits the linear separability of credit data more effectively than majority-vote distance-based classification.

Marketing context: In lead scoring for financial products, credit-like features (income tier, employment stability) often have one dominant signal — a pattern where Chebyshev-style 'max-feature' logic can outperform Euclidean distance in small, well-structured datasets.

Loan Default — KNN Analysis¶

Best configuration: k=29, Manhattan distance (CV AUC: 0.6234 → Test AUC: 0.6254)

Performance ceiling confirmed: AUC 0.6254 — consistent with the 0.62–0.69 range observed across all Weeks 1–6. The dataset's missing payment history and credit bureau signals impose a Bayes error floor that no model family can overcome with the available features.

Why Manhattan wins: L1 distance (absolute differences, no squaring) outperforms Euclidean and Chebyshev. Loan features likely contain outliers in income and loan amount; Manhattan's linear error propagation handles these more robustly than L2's squared amplification.

Smallest Delta AUC (−0.0689 vs Wk6 RF): KNN's inability to weight features is relatively less damaging here — no single feature dominates the loan default signal either, so the equal-weight assumption hurts less than in the fraud dataset.

Marketing context: Loan default patterns mirror churn in subscription services — when no single behavioral feature dominates, ensemble tree models consistently outperform distance-based methods due to their ability to capture non-linear feature interactions.

Common finding across all three datasets: optimal k was 29 in every case, with AUC still rising at the boundary of the search range. This indicates KNN favors larger neighborhoods on these datasets — a sign that local structure is noisy and averaging over more neighbors reduces variance.

Why parametric and tree models dominate: KNN's three structural limitations (no feature weighting, sensitivity to class imbalance, curse of dimensionality) are all present in this feature space. The algorithm's strength — zero training time and interpretable neighbor-based logic — does not compensate in this high-dimensional, imbalanced classification context.

One deployment advantage KNN retains: with a pre-built index, inference is sub-millisecond per transaction — a latency profile that parametric models trained on 1.3M rows cannot match at serving time.
